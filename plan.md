# 국내주식 개인 자동매매 개발 계획 (KIS Open API)

작성일: 2026-03-23
최종 업데이트: 2026-03-23

## 1) 목표
- 국내주식 개인용 자동매매 서버를 구축한다.
- KIS Open API를 사용해 시세 조회, 주문, 정정/취소, 체결 확인까지 자동화한다.
- 실거래 전 모의환경과 리스크 통제 체계를 먼저 완성한다.

## 2) 확정된 기술/API 선택
- 브로커 API: **KIS Open API**
- 사용 범위: **국내주식 + 개인용**
- 개발 환경: macOS 개발 가능 (REST/WebSocket 기반)

## 3) 핵심 원칙
- 단일 필승 알고리즘은 없다는 전제로 설계한다.
- 수익보다 먼저 **손실 통제**를 시스템에 강제한다.
- 과최적화 방지를 위해 백테스트/워크포워드/페이퍼트레이딩 단계를 분리한다.

## 4) 초기 전략(1차)
- 전략 유형: 추세추종 기반 (단순, 재현성 높은 규칙)
- 종목군: 유동성 높은 대형주/ETF 중심
- 예시 규칙:
  - 진입: 20일 고점 돌파 + 60일 추세 필터
  - 청산: ATR 기반 손절 또는 고정 손절(-2%~-3%)
  - 포지션 크기: 1회 손실을 총자산의 0.5% 이내로 제한
  - 킬스위치: 일손실 -2% 도달 시 당일 자동매매 중지

## 5) 서버 구성(최소 기능)
- `market-data`: 시세 수집/정규화
- `strategy-engine`: 매수/매도 신호 생성
- `risk-manager`: 주문 허용/차단, 한도 검증
- `execution`: 주문/정정/취소 및 중복 주문 방지
- `portfolio`: 잔고/포지션/손익 계산
- `trade-journal`: 주문/체결/에러/의사결정 로그 저장

## 6) 개발 단계
1. KIS 인증/토큰 발급 및 계좌 연동 확인
2. 시세 조회 + 주문 단위 모듈 구현
3. 리스크 매니저(최대 손실/최대 보유종목/중복 주문 차단) 구현
4. 전략 엔진 1종 연결 (추세추종)
5. 모의투자 장중 검증
6. 실거래 소액 전환

## 7) 운영 체크리스트
- 장 시작 전 토큰/세션 상태 점검
- 주문 실패 재시도 정책(횟수/간격/중복 방지 키)
- 체결 이벤트 기준으로 상태 확정 (주문 응답만 신뢰하지 않음)
- 장애 시 강제 중지 스위치 제공
- 모든 주문 의사결정 로그를 남겨 사후 분석 가능하게 유지

- 주문/체결/시세 TR별 필드 매핑 표준화
- 오류 코드/제한 정책(호출 한도, 장중 에러) 반영

2. 장 운영 상태머신 추가
- 한국장 캘린더/시간대 기준으로 pre-open/open/close 상태 분리
- 장 종료 시 주문 동기화/정산/리포트 루틴 추가

3. 실행/복구 안정성 강화
- 주문 실패 재시도 횟수/간격 정책 고정
- idempotency 키와 중복방지 키 운영 규칙 고도화
- WS 단절/지연/중복 체결 이벤트 시나리오 테스트 확장

4. 모의투자 운영 검증
- 5영업일 연속 장중 모의운영
- 합격 기준:
  - 치명 오류 0
  - 중복 주문 0
  - 리스크 규칙 위반 0

5. 실거래 전환 준비
- 실거래 사전점검 API 및 체크리스트 제공
- 기본은 `LIVE_MODE=false` 유지, 수동 승인 절차 후 소액 전환
